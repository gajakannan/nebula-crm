# Inception Requirements

Generic completion criteria for Phase A (Product Manager Mode).

## Purpose

This checklist defines what “complete” means for Phase A. Project-specific criteria live in `planning-mds/`.

---

## Phase A Deliverables

All Phase A work is documented in `planning-mds/INCEPTION.md` sections 3.1–3.5, plus any feature/story files referenced there.

### Section 3.1: Vision & Non-Goals

- Vision statement (2–3 sentences)
- 3–5 measurable success criteria
- 5–10 explicit non-goals

### Section 3.2: Personas

- Minimum 3 personas
- Each persona includes goals, pain points, JTBD

### Section 3.3: Epics / Features

- All core epics/features listed
- Each has business objective and scope boundaries

### Section 3.4: MVP User Stories

- At least one complete vertical slice
- Stories include acceptance criteria, edge cases, and out-of-scope items

### Section 3.5: Screen Specifications

- Key screens listed
- Each screen has purpose, primary data, and actions

---

## Phase A Completion Checklist

- [ ] Vision and non-goals documented
- [ ] Personas defined
- [ ] Epics/features defined
- [ ] Stories complete with acceptance criteria
- [ ] Screens specified
- [ ] No TODOs remain
